## 1. `satisfies` over `as`

```ts
const config = {
  theme: 'dark',
  lang: 'en',
} satisfies Record<string, string>
```

`satisfies` validates the shape but keeps the inferred type narrow. `as` lies to the compiler — `satisfies` doesn't.

## 2. Branded types

```ts
type UserId = string & { readonly __brand: 'UserId' }
const toUserId = (s: string): UserId => s as UserId
```

Prevents accidentally passing a raw string where a `UserId` is expected. Zero runtime cost.

## 3. Template literal types

```ts
type EventName = `on${Capitalize<string>}`
// valid: 'onClick', 'onChange' — invalid: 'click', 'change'
```

## 4. `infer` in conditional types

```ts
type Awaited<T> = T extends Promise<infer U> ? U : T
```

Extract the resolved type of any Promise without a utility type import.

## 5. `const` assertions for exhaustive maps

```ts
const LANGS = ['ts', 'py', 'vue'] as const
type Lang = (typeof LANGS)[number] // 'ts' | 'py' | 'vue'
```

Add a value to the array and the union updates automatically.
